# ⚡ MeterMind — AI-Powered Smart Grid Intelligence Platform

> Real-time energy analytics, loss fingerprinting, and demand forecasting for modern smart grids — powered by Machine Learning and a live synthetic meter simulator.

🌐 **Live Demo:** [meter-mind.vercel.app](https://meter-mind.vercel.app) | **Backend API:** [meter-mind.onrender.com](https://meter-mind.onrender.com)

---

## 🧠 What is MeterMind?

MeterMind is an end-to-end smart grid intelligence platform that processes thousands of synthetic smart meter readings in real time. It applies machine learning to detect energy loss patterns and forecast demand — giving utilities and grid operators a live, AI-driven window into their network.

The system is split into three tightly integrated components:

| Layer | Tech | Purpose |
|---|---|---|
| **AI Engine** | Python, scikit-learn / ML models | Trained models for loss fingerprinting & demand forecasting |
| **Backend** | Python, FastAPI | REST API server + live smart meter data simulator |
| **Frontend** | Vanilla HTML, CSS, JavaScript | Real-time dashboard with charts and analytics |

## ✨ Key Features

- **⚡ Live Smart Grid Simulator** — Generates thousands of synthetic smart meter readings every few seconds, continuously feeding the backend with realistic grid data.
- **🔍 Loss Fingerprinting** — ML model that identifies and classifies energy loss patterns (e.g. technical losses, non-technical losses / theft) across the grid.
- **📈 Demand Forecasting** — Predicts upcoming energy demand based on historical meter patterns, enabling proactive grid management.
- **📡 Real-Time Dashboard** — A zero-dependency Vanilla JS frontend that polls the live API and renders analytics with minimal latency.
- **☁️ Cloud-Ready** — Backend deployed on Render, frontend on Vercel; works out of the box without any local setup.

## 🚀 Local Setup & Run Guide

Follow these steps to run the full MeterMind stack on your machine.

### Prerequisites

- **Python 3.9+** installed
- **VS Code** with the [Live Server extension](https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer) *(optional — Python fallback available)*

---

### Step 1 — Start the AI Backend & Live Simulator

The backend spins up a FastAPI server and launches a background simulator that continuously POSTs synthetic meter data payloads.

```bash
# 1. Navigate into the backend folder
cd backend

# 2. Install Python dependencies
pip install -r requirements.txt

# 3. Start the server + simulator together
python run.py
```

⏳ **Wait ~15 seconds.** You will see logs confirming:
- FastAPI server is running at `http://localhost:8000`
- Live Simulator is successfully posting data payloads

> Leave this terminal window open and running.

---

### Step 2 — Start the Frontend Dashboard

The frontend is pure HTML + CSS + Vanilla JS — no `npm install` required.

**Option A — VS Code Live Server (recommended):**
1. Open the `frontend/` folder in VS Code.
2. Right-click `index.html` → **"Open with Live Server"**.

**Option B — Python HTTP server:**
```bash
# Open a new terminal window
cd frontend
python -m http.server 3000
```
Then open your browser at `http://localhost:3000`.

---

### Step 3 — Point the Frontend at Your Local Backend

> ⚠️ **IMPORTANT:** By default, the frontend is configured for production and talks to the live cloud backend at `https://meter-mind.onrender.com`.

To connect it to your locally running backend instead:

1. Open `frontend/loss_fingerprinting.js`
2. Open `frontend/demand_forecast.js`
3. In **both files**, find and replace:

```
https://meter-mind.onrender.com  →  http://localhost:8000
```

4. Refresh the browser tab.

> **Skipping this step is fine** — the dashboard will still work perfectly using the live cloud data instead of your local simulator.

## 🔌 API Endpoints

Once the backend is running, the following endpoints are available at `http://localhost:8000`:

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/` | Health check / API root |
| `POST` | `/ingest` | Accepts meter reading payloads (used by the simulator) |
| `GET` | `/loss` | Returns loss fingerprinting analysis results |
| `GET` | `/forecast` | Returns demand forecast predictions |

Interactive API docs (auto-generated by FastAPI) are available at:
```
http://localhost:8000/docs
```

## 🤖 AI / ML Models

The `ai_engine/` directory contains the trained machine learning models and any training scripts used to build them.

- **Loss Fingerprinting Model** — Classifies meter readings into loss categories. Likely a supervised classifier (e.g. Random Forest / XGBoost) trained on labeled synthetic meter data.
- **Demand Forecast Model** — Predicts future energy demand from time-series meter patterns. Likely a regression or time-series model (e.g. LSTM / LightGBM).

Models are loaded by the FastAPI backend at startup and served via the `/loss` and `/forecast` endpoints.
